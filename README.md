# SQL Data Cleaning Project: Layoffs Dataset

## Project Overview

This project focuses on cleaning and preparing a real-world layoffs dataset using SQL. The objective was to improve data quality and transform the raw dataset into an analysis-ready format.

## Skills Demonstrated

* Data Cleaning
* SQL Window Functions
* Data Validation
* Handling Missing Values
* Data Standardization
* Duplicate Removal

## Cleaning Steps

1. Identified and removed duplicate records.
2. Standardized company, industry, and country names.
3. Converted blank values to NULLs.
4. Handled missing data appropriately.
5. Removed invalid records.
6. Verified data integrity after cleaning.

## Tools Used

* MySQL
* GitHub

## Dataset

The dataset contains information on company layoffs, including company names, industries, locations, funding raised, layoff counts, layoff percentages, and dates. The data was cleaned and transformed to improve consistency, accuracy, and usability for analysis.

## Project Files

* `data_cleaning.sql` – SQL script containing all data cleaning operations.
* `layoffs.csv` – Original dataset used for the project.
* `README.md` – Project documentation.
* `images/` – Screenshots of key cleaning steps and results.

## Key Outcomes

* Improved overall data quality and consistency.
* Removed duplicate and invalid records.
* Standardized categorical fields for accurate analysis.
* Converted missing and blank values into a usable format.
* Produced a clean dataset ready for exploratory data analysis and visualization.
